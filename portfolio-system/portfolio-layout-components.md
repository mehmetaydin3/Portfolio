# Portfolio Layout Components

## Purpose

Defines reusable layout components used to construct portfolio case study pages.

These components ensure:

• consistent layout patterns  
• scalable page structure  
• visual storytelling clarity  

Each component represents a structural building block.

---

# Component: Hero Section

Purpose: Introduce the project and establish visual tone.

Structure:

Headline  
Subheadline  
Metadata row  
Hero visual

Metadata example:

Role · Timeline · Platform · Screens

Hero visuals should be large and immersive.

---

# Component: Context Block

Purpose: Provide project background.

Layout:

Centered text block.

Max width:

700px

Optional supporting visual.

---

# Component: Opportunity Section

Purpose: Frame the problem as a product opportunity.

Layout:

Text + pull quote.

Example:

> “I don't know what guidance applies to me anymore.”

Pull quotes should be visually prominent.

---

# Component: Architecture Section

Purpose: Demonstrate systems thinking.

Layout:

Text + diagram.

Diagram types:

• routing system  
• platform architecture  
• marketplace flow  
• modular components  

Diagrams should remain simple.

---

# Component: Feature Card Grid

Purpose: Highlight key product decisions.

Layout:

Responsive grid.

Example:

[ Feature Card ] [ Feature Card ]  
[ Feature Card ] [ Feature Card ]

Each card contains:

Feature title  
Short description  
Impact metric

---

# Component: Feature Spotlight

Purpose: Highlight one meaningful feature.

Layout:

Two-column split.

Left:

Explanation.

Right:

Large UI visual.

On mobile this collapses vertically.

---

# Component: UI Showcase

Purpose: Display finished product.

Layout:

Large edge-to-edge visuals.

Minimal text.

Screens should be presented in sequence.

---

# Component: Metrics Grid

Purpose: Present outcomes.

Layout:

3–4 metric cards.

Example:

+41%  
Quiz start rate

87%  
Completion rate

+42%  
Scroll depth

Metrics should be visually prominent.

---

# Component: Reflection Section

Purpose: Close with strategic insight.

Layout:

Single editorial paragraph.

Concise and thoughtful.

Avoid generic lessons.

---

# Layout Rhythm

Pages should follow this rhythm:

Hero  
Context  
Opportunity  
Architecture  
Feature Cards  
Spotlight Feature  
UI Showcase  
Metrics  
Reflection
